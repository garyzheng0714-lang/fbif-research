# 质量检查 + 审查 详细规范

## 检查Agent（质量守门人）

用户启动调研后，不需要任何人工确认，检查agent自动在每个步骤完成后执行核验。

### 每一步的检查项

**第1步——搜索覆盖：**
- 每个模块的URL数是否达到最少要求？
- 是否覆盖了品牌母语/英文/中文三种语言？
- 是否有遗漏的重要搜索维度（创始人演讲、行业报告等）？
- 不达标 → 自动补搜

**第2步——抓取完整性：**
- 逐个打开source文件，检查原文是否完整（≥500字、未截断、非导航噪音）
- blocked的URL是否真的穷尽了7级回退？
- 总字数是否达标？
- 不达标 → 自动让抓取agent补抓

**第3步——翻译质量：**
- 每篇外文是否都有"## 中文翻译"段落？
- 翻译字数是否≥原文50%？
- 抽查翻译是否通顺、是否有遗漏段落？
- 不达标 → 自动让翻译agent补翻

**第4步——最终核验：**
- HTML能否正常打开？
- 全文中是否所有来源都展示了完整正文？
- ZH/EN切换是否正常？
- "复制本模块"按钮是否在模块标题右边？
- 总字数是否≥82,000，中文≥30,000？
- OSS是否上传成功？公网链接是否可访问？
- 抓不到的来源是否在报告末尾"待人工查看"区域列出？

### 执行方式

检查用脚本，不靠AI判断：
```bash
python3 scripts/check_quality.py {artifact_root} --step 1|2|3|4|all
```
退出码：0=通过，1=有问题。报告写入 `quality-report.json`。

## 第5步：审查Agent

部署上线后运行：
```bash
python3 scripts/audit_report.py {artifact_root}
```

### 审查项目

1. **HTML结构完整性** — DOM有没有被打碎（body直接子元素数量异常）
2. **标签泄露** — 正文中有没有未转义的原始HTML标签
3. **噪音内容** — 有没有残留的广告、Cookie横幅、社交按钮
4. **空内容检测** — 有没有中文内容为空的来源卡片
5. **模块覆盖度** — 模块数是否≥5
6. **两级目录** — toc-item-h3是否存在
7. **文件大小** — HTML是否过大影响加载

### 审查→修复循环

不通过时：
1. 读取 `audit-report.json` 中的 issues 列表
2. 标签泄露 → 重新运行 `clean_content.py` + `assemble_single.py`
3. 噪音内容 → 检查来源文件，补充清洗规则
4. 空内容 → 检查来源文件，补抓或标记为"待人工查看"
5. 修复后重新 `deploy_oss.py` + 再次审查
6. 最多循环3次

### 交付格式

```
调研完成：
- 公网链接：http://oss.garyzheng.com/{brand}/index.html
- 来源数量：N篇
- 总字数：N / 中文N
- 审查结果：通过
- 待人工查看：N篇（见报告末尾）
```
