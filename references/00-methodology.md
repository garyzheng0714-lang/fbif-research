# 调研核心方法论（全局参考文件）

**本文件包含完整的调研原则、文章创作理念、输出规范。所有模块执行前必须阅读本文件。**

## ⚠️ 调研总体量目标：10万字左右

调研资料总量目标在10万字左右（中文字数≥5万）。这个体量来自：每个模块的原始资料全文+中文翻译。特别是模块八（深度访谈）单独就应该有3-10万字。如果最终产出总字数不到5万字，说明调研深度严重不足，必须返回补充。

---

## 一、执行形态原则

### 1. 文件优先，不把10万字资料塞进聊天

长任务默认应把原文、翻译、来源索引、阶段报告写入输出目录，由文件承载大体量内容，聊天只保留简短进度汇报、阻塞说明和产物路径。

### 2. 阶段是检查点，不是确认点

调研可以分阶段推进，但默认应自动续跑。除非出现品牌歧义、登录/付费墙、目录不可写、模板缺失等真正阻塞，否则不要在每个阶段结束时要求用户确认继续。

### 3. 每个来源都要可追踪、可恢复

必须维护来源清单和状态文件。每发现一个来源就登记状态；每完成一次抓取就更新状态；中断后先读取状态文件再继续，避免漏项和重复劳动。

## 二、信息获取原则

1. **一次调研，获取所有信息** — 不遗漏任何模块
2. **完整的原始的品牌发展历史信息** — 保留所有原始资料
3. **推动事物发展的关键动作** — 识别关键决策和行动
4. **保留所有信息的来源** — 来源链接、来源平台名称、时间
5. **优先母语调研** — 先用品牌母国语言调研（日本品牌先查日文，德国品牌先查德文），这是核心资料。其次英文、中文。但需结合实际判断：如日本公司的子品牌在美国，优先调研英文。

## 三、查资料原则

- **底线信息**：用事实和数据回答，不需要分析，简洁明了
- **深度分析**：要有具体的信息、数据、案例支撑
- **禁止硬写**：查不到有深度的信息，底线信息即可，不得编造分析
- **未找到的信息**：明确标注"未找到公开信息"，不得编造

## 四、翻译原则

- 保留原文全文的信息基础上进行中文翻译（要地道不要翻译腔）
- 如果原文是中文就不用翻译

## 四.五、内容获取引擎：Jina Reader

**Jina Reader 是调研的核心抓取工具。** 通过 `https://r.jina.ai/{url}` 将任何网页转换为干净的 Markdown，免费、无API密钥、能处理JS渲染和反爬虫。

**在 Claude Code 中使用：**
```
WebFetch URL: https://r.jina.ai/{target_url}
Prompt: "Extract the complete article text. Return every paragraph of the main content."
```

**脚本方式：**
```bash
python scripts/jina_fetch.py "{url}" "sources/src-{xxxx}.md"
```

**7级回退链路（当 Jina Reader 失败时）：**
1. Jina Reader 重试
2. 直接 WebFetch 原URL
3. 搜索转载 — `"{文章标题}"` 找其他平台全文
4. Wayback Machine — `https://r.jina.ai/https://web.archive.org/web/{原URL}`
5. 多次搜索拼接 — 5-10次搜索可拼接80-95%内容
6. PDF版本 — `"{文章标题}" filetype:pdf`
7. 作者其他渠道 — LinkedIn/Medium/个人博客

**视频/播客回退链路：**
- YouTube：搜索 transcript → 第三方字幕提取 → 演讲文字版搜索
- 播客：官网 transcript → RSS show notes → 嘉宾其他渠道内容

**只有所有回退链路都实际尝试并失败后，才能标记为 blocked。**

## 五、权威来源、数据严谨

- 数据应来源于官方或权威渠道，观点要有依据，不凭空创作
- 所有信息必须是官网、权威来源，不能有凭空创作的任何信息

## 六、文章创作核心理念

调研时需时刻牢记以下理念，它们决定了调研方向和深度：

### 1. 观点原则
参考Reuters与Associated Press的编辑规范，观点必须建立在可核实事实与逻辑推理之上，并清晰区分事实、分析与立场，避免主观臆断。

### 2. 数据原则
所有数据必须来源透明、可追溯、方法清晰，并经过交叉验证，结论不得超出数据本身所能支持的范围。

### 3. 人物与决策还原原则
大多数的创新，背后一定是有关键的人物的推动。识别关键决策者——如果存在关键人物，必须交代是谁在推动、承担与拍板。必须解释决策当时所处的环境与信息限制。避免retrospective bias——不能用结果倒推合理性。

### 4. 应当避免的创作问题

很多时候讲商业故事成功都太顺了，理所当然。好像一个点子是突然想出来了（品牌可能会也会回避模仿抄袭），实际上很多成功，是因为看到别人先成功，自己再跟进。例如，Charles & Alice的"无糖"，如果不提Hero，逻辑会变成："一家小公司突然想明白了不加糖，于是成功了。"

很多文章默认就不提失败和失败的部分，比如Mogu Mogu在韩国坐了7年冷板凳，根本原因就是没有跟当地合作，渠道进不去，后来想明白了这事才有进展。这种此前错误的决策不提，此后的成功都会被理所当然，或者是被其他原因所解释。

所有的决策，也极少是理所当然就成的。如果什么都决策对比了，要么没有怎么成功，要么就是回避。阿里巴巴的成长起来前有无数错误，成长起来后也是无数的错误，这个才是最真实的。

### 5. 商业成功的前提是"消费者愿意买"

商业成功它的答案不在商业的故事和数据里，但我们要理解它，就要去消费者那里找答案。

几乎很少人去站在消费者的角度去理解，为什么愿意买。对于我们的读者而言（我们的读者是一个个真实的行业里的人，他们都是想要从中学习，得到启发，最后真正动手去做），如果脱离了最本质、最核心的消费者愿不愿意买，为什么愿意买，所有的分析都没有意义，因为大家学不会，或者最终只是去"抄"个形，甚至学错。

商业成功最核心的前提是"消费者愿意买"，有了这个前提，才有后续的渠道、供应链、生产、品牌化等等一切。就像可口可乐，如果不好喝，它铺向全世界的所有角落都没有用。但很可能大家都习惯了去说，可口可乐成功的原因是因为"无处不在"和"便宜"。但它首先得好喝，还要上瘾。

### 6. 错误和失败才是成功最真实的一部分

创业和创新更加真实的其实是大量、长期的错误和失败，这是创业和创新过程中最真实的部分，也是世界上绝大多数成功者背后最真实的部分，辉煌时刻只是少数，哪怕是乔布斯主导开发的大部分产品都是失败的（最多1/5是成功），任何一个产品和人物成功的背后，都有无数的试错，大多数天才也要走过这些弯路才更有可能取得成功，只是大多数品牌的故事不会去写，大多数人也都不会去写。

### 7. 抓住成功的重点，别只看到表面

有一个非常典型的案例，就是日本的糖果品牌甘乐。很多年前就接触这个案例，第一次听到时，大家都在说它创新、创新，直到自己深入了解才发现，他除了成为硬糖的冠军以外，很重要的是同时抓住了软糖的趋势。但这不是它一年取得15亿营收的根本原因，因为只靠产品创新，做不到。

所有人都忽略了一点是，创新只是它的从0到1，他从1到100是跟三菱的合作，让他的渠道可以瞬间铺向全国，直到今天，它93%的销售额仍然是来自三菱，这才是他规模化的根本原因。如果忽略了这一点，分析文章里不写，我们只学他的产品创新，那我们的客户学习了可能也不知道怎么才能取得商业上的成功。

## 七、来源标注原则（全局，所有模块适用）

**所有10个模块的每一条关键信息都必须标注来源。** 不只是模块一和模块八——品牌历史、关键数据、品类背景、4P、包装、消费者评价、最新动态中的每一个事实、数据、引用都必须附出处。

**来源信息必须完整，每次都写全，禁止偷懒：**
- ❌ 禁止使用"同上""同前""见上文""来源同上"等省略写法
- ✅ 每条来源都必须包含：来源标题、平台/媒体名、发布时间、原文链接
- ✅ 如有作者/被采访人，也要标注

无来源的信息 = 不可信的信息 = 不能放进文章的信息。如果找不到来源，标注"未找到公开信息来源"。

## 八、命名与语言铁律

### 人名处理
非世界级名人（如Elon Musk），严禁音译。必须保留英文原名（如：保留Kara Goldin，禁止写卡拉·高汀）。中国人、华人、日本、韩国人名，也使用中文名。

### 品牌处理
优先级为 官方中文名 > 官方英文名 > 原文。若无官方中文名，严禁自行意译或音译，直接保留英文。

### 小语种（日/韩/俄/泰）
若无官方中/英文名，必须保留原文的基础上备注英文和中文，禁止强行翻译，并在文末清单列出。

---

## 九、格式规范

- 数字与中文单位之间不得出现任何空格（如必须写成"1100亿日元""40年""1/3国民"，禁止"1100 亿日元""40 年"）
- 英文单词与中文之间也不要有空格（如"它的品牌名叫Charles & Alice"，不是"它的品牌名叫 Charles & Alice"）
- 英文符号 &、|、- 等在中文里出现时要确保前后有空格
- 去掉所有不必要的格式，比如多余的*符号
