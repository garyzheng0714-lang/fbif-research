# 模块一：核心官方资料

## ⚠️ 字数红线：≥10,000字（含原文+翻译）。不到10,000字 = 不达标，必须补充。

## 目标

获取品牌/公司最权威的一手信息来源，为后续所有模块提供事实基础。

## ⚠️ 本模块执行铁律

### web_fetch 强制要求

本模块的核心价值在于**完整的原始信息**。执行分两步：

1. **web_search 发现页面** — 搜索找到官网、百科等页面的URL
2. **web_fetch 获取全文** — 对每个找到的URL，使用 web_fetch 抓取完整页面内容

**严禁用 web_search 的搜索摘要片段代替页面全文。** 搜索摘要通常只有几句话，而一个官网"关于我们"页面可能有几千字——这些内容必须完整获取。

### "完整"的定义

- 官网页面：该页面上所有可见文字内容，包括子栏目、时间线、引用语等。web_fetch返回的完整内容必须全部保留，不做任何删减或概括
- 百科条目：条目正文全部内容（不含参考文献列表和导航栏）
- 深度介绍文章：文章正文全部内容

**⚠️ 保留原文 + 全文翻译：** 每个外文页面的内容，必须同时在HTML中保留原始语言全文和完整中文翻译。翻译是在原文基础上的"加法"，不能用翻译替代原文。

### web_fetch失败的补救（官网类）

官网页面被拦截时，按以下优先级尝试：
1. **直接重试** — 偶尔因超时失败
2. **Wayback Machine** — `web_fetch("https://web.archive.org/web/{官网URL}")` 存档版本通常可以抓到
3. **搜索引擎缓存** — web_search `cache:{URL}`
4. **搜索相同内容的其他页面** — 官网内容经常被行业媒体、百科引用或转载
5. **多次搜索拼接** — 用页面上的关键短语搜索，拼接还原

**不能因为官网被拦截就跳过官方资料模块。**

### 体量预期

本模块完成后的文档应有 **1-3万字**（含原文+中文翻译）。如果产出不到5000字，说明没有执行 web_fetch，需要返回补充。

### 示例

以 Taifun Tofu 为例：
- 搜索找到 `https://www.taifun-tofu.de/en/magazine/the-history-of-taifun-tofu`
- 必须 web_fetch 这个URL，获取页面上关于品牌历史的完整内容（可能有数千字）
- 把完整原文录入文档 + 完整中文翻译
- ❌ 错误做法：只用搜索摘要的两三句话概括

## 调研内容

### 1. 官网信息（必查）

**必须 web_fetch 的页面清单：**

- 公司/品牌官网首页
- "关于我们" / "About" / "Über uns" 页面
- 公司/品牌历史页面（如果单独存在）
- 企业理念、愿景、使命页面
- 产品总览页面
- 新闻/媒体页面（最新几条）

**执行步骤：**
1. web_search 找到官网域名
2. web_fetch 首页，浏览确认上述页面的URL
3. 逐个 web_fetch 获取完整内容
4. 录入原文全文 + 中文翻译

### 2. 百科平台（必查）

默认必须包含以下百科平台：

| 品牌来源国 | 必查百科平台 |
|------------|-------------|
| 所有品牌 | 维基百科（优先母语版本） |
| 中国品牌 | 百度百科 |
| 韩国品牌 | Namu Wiki（优先）、维基百科韩语版 |
| 日本品牌 | 维基百科日语版 |
| 德国品牌 | 维基百科德语版 |

**执行步骤：**
1. web_search 找到百科条目URL
2. web_fetch 获取条目完整内容
3. 录入原文全文 + 中文翻译

### 3. 深度历史介绍文章

查找权威渠道对该品牌/公司的深度介绍文章。

**执行步骤：**
1. web_search 搜索品牌深度介绍/专题报道
2. 找到有价值的文章URL
3. web_fetch 获取文章完整内容
4. 录入原文全文 + 中文翻译

## 调研语言优先级

1. 品牌母国语言（核心资料）
2. 英文
3. 中文

## 输出格式

### 资料汇总表（放在文档开头）

| 原文标题 | 平台来源 | 发布时间 | 作者 | 一句话中文摘要 | 链接 |
|----------|----------|----------|------|---------------|------|

- 原文标题：保留原始语言
- 平台来源：写平台名称（如"官网""维基百科德语版"）
- 发布时间：尽量精确，未知则填"—"
- 作者：未知则填"—"
- 一句话中文摘要：简要说明该资料的核心内容

### 官网内容（汇总表之后，逐页录入）

对每个官网页面：

```
=== 页面标题 ===
来源链接：{URL}

【原文全文】
{web_fetch 获取的完整页面内容，一字不删}

【中文翻译】
{完整中文翻译，地道表达，不翻译腔}
```

### 百科内容

对每个百科条目：

```
=== 条目标题 ===
来源链接：{URL}
平台：{百科平台名称}

【原文全文】
{web_fetch 获取的完整页面内容，一字不删}

【中文翻译】
{完整中文翻译，地道表达，不翻译腔}
```

### 深度介绍文章

对每篇文章：

```
=== 文章标题（原文） ===
中文标题：{翻译}
来源链接：{URL}
平台/媒体：{名称}
发布时间：{日期}
作者：{姓名}

【原文全文】
{web_fetch 获取的完整正文，一字不删}

【中文翻译】
{完整中文翻译，地道表达，不翻译腔}
```

## 质量自检

1. ☐ 是否对每个URL都执行了 web_fetch？（不是只用搜索摘要）
2. ☐ 官网的关于/历史/理念页面是否全部获取？
3. ☐ 百科条目是否完整获取？
4. ☐ 所有原文是否完整保留，没有删减？
5. ☐ 中文翻译是否完整且地道？
6. ☐ 文档总字数是否达到1万字以上？如果不到，大概率遗漏了内容。

## 输出文件

文件名：`1-官方资料-{品牌名}.docx`
