# HTML报告规范（v10.2）

**禁止自行编写HTML。** 必须使用模板 + 组装脚本。

组装命令：
```bash
python3 scripts/assemble_single.py {artifact_root}
```

## 页面布局

```
┌─────────────────────────────────────────────────────┐
│  [复制全部调研资料]              [下载] [分享]        │  ← 固定右上角
├─────────────────────────────────────────────────────┤
│                                                     │
│  ┌──────────────────────────────────────────────┐   │
│  │ 目录区域（各模块标题，标题右边有"复制"按钮）    │   │
│  └──────────────────────────────────────────────┘   │
│                                                     │
│  主内容区                                            │
│  ────────                                           │
│  [模块标题] ──────────────── [复制本模块]             │
│  来源1标题(超链接)  域名 · 字数                       │
│  全文内容...                                         │
│  来源2标题(超链接)  域名 · 字数                       │
│  全文内容...                                         │
│                                                     │
│                               [ZH|EN] [↑]           │  ← 右下角浮动
└─────────────────────────────────────────────────────┘
```

## 关键布局规则

1. **没有"摘要"Tab** — 只展示全文，不做摘要
2. **"复制全部调研资料"** — 固定在右上角，与下载/分享按钮并排
3. **"复制本模块"** — 放在模块标题的右边（同一行），不是在目录下方
4. **语言切换 [ZH|EN]** — 右下角浮动按钮，与回到顶部按钮并排
5. **去掉所有不必要的徽章和标签** — 保持干净
6. **全文不截取** — 原文多长就展示多长，一个字不落
7. **模块折叠** — 默认只展开第一个模块，点击标题展开/折叠，减少初始渲染量
8. **无动画** — 目录跳转使用 `behavior: 'auto'`，不用 smooth scroll

## 全文内容结构

- 按模块分组展示所有来源的完整正文
- 模块标题行：模块名 + 来源数 + "复制本模块"按钮（右对齐）
- 每个来源：标题（超链接）+ 域名 + 字数
- 正文直接展示，没有外层卡片边框
- ZH模式显示中文翻译，EN模式显示原文
- 来源标题必须带原始超链接（可点击跳转）

## 两级目录 + 语言联动

- 左侧目录分两级：大标题是模块中文名，子目录是该模块下每篇文章的标题
- 当前阅读的模块自动展开子目录，其他模块的子目录折叠
- **标题和目录必须跟随ZH/EN切换**：ZH模式显示中文标题，EN模式显示原文标题
- 组装脚本从翻译内容中提取中文标题，存为 `data-title-zh` 属性；原文标题存为 `data-title-en`
- 语言切换时，JS同时更新文章标题和TOC子目录的文本

## 图片渲染

source文件中的图片 `![alt](url)` 在HTML中渲染为 `<img>` 标签，图片直接显示在全文中。

## 安全规则

1. **HTML转义** — `process_inline` 对所有原文内容做 `<` → `&lt;` 转义
2. **URL协议过滤** — 图片和链接禁止 `javascript:`、`data:`、`vbscript:`
3. **外链安全** — `<a>` 标签自动加 `rel="noopener noreferrer"`
4. **CSP头** — 模板内置 Content-Security-Policy meta标签
5. **内容清洗** — `clean_content.py` 在组装前去除可疑内容
